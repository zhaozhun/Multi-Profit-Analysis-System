# 指标数据测试数据说明

## 数据结构概述

### 1. 指标汇总表 (indicator_summary)

存储各指标的月度汇总数据，用于指标数据页面展示。

| 字段 | 说明 | 示例 |
|------|------|------|
| period | 账期月份 | 2026-01 |
| indicator_code | 指标编码 | INTEREST_INCOME |
| indicator_type | 指标类型 | ATOMIC |
| business_line | 业务条线 | ASSET/LIABILITY |
| calc_value | 汇总值（万元） | 12568.50 |

### 2. 费用分摊结果表 (cost_allocation_result)

存储费用分摊到账户的明细数据。

| 字段 | 说明 | 示例 |
|------|------|------|
| batch_no | 批次号 | BATCH001 |
| period | 账期月份 | 2026-01 |
| cost_code | 费用编码 | DATA_FEE |
| cost_name | 费用名称 | 数据使用费 |
| original_amount | 原始金额 | 954.20 |
| target_type | 目标类型 | ACCOUNT |
| target_code | 目标编码 | ACC001 |
| target_name | 目标名称 | 账户1 |
| allocated_amount | 分摊金额 | 125.50 |
| allocation_method | 分摊方法 | BY_DEPT_RATIO |

### 3. 费用类型配置表 (cost_type_config)

存储费用类型的基本信息。

| 字段 | 说明 | 示例 |
|------|------|------|
| cost_type_code | 费用类型编码 | DATA_FEE |
| cost_type_name | 费用类型名称 | 数据使用费 |
| parent_code | 父级编码 | OP_COST |
| level | 层级 | 2 |
| description | 描述 | 数据服务和数据采购费用 |
| default_algorithm | 默认算法 | BY_DEPT_RATIO |
| default_factor | 默认因子 | WORK_HOUR |
| status | 状态 | ACTIVE |

### 4. 费用原始记录表 (cost_actual_record)

存储费用的原始发生数据。

| 字段 | 说明 | 示例 |
|------|------|------|
| period | 账期月份 | 2026-01 |
| cost_code | 费用编码 | DATA_FEE |
| cost_name | 费用名称 | 数据使用费 |
| cost_type | 费用类型 | OPERATING |
| amount | 金额 | 311.40 |
| dept_code | 部门编码 | DEPT001 |
| org_code | 机构编码 | ORG001 |
| vendor | 供应商 | 数据服务商A |
| invoice_no | 发票号 | INV202601001 |
| occurrence_date | 发生日期 | 2026-01-15 |
| description | 说明 | 1月数据使用费-零售部 |
| status | 状态 | CONFIRMED |

## 使用方法

### 1. 执行数据插入脚本

```bash
mysql -u root -p multi_profit < 数据库脚本/insert_test_data.sql
```

### 2. 验证数据

```sql
-- 查看指标汇总数据
SELECT * FROM indicator_summary WHERE period = '2026-01';

-- 查看费用分摊结果
SELECT * FROM cost_allocation_result WHERE period = '2026-01';

-- 查看费用类型配置
SELECT * FROM cost_type_config;

-- 查看费用原始数据
SELECT * FROM cost_actual_record WHERE period = '2026-01';
```

- 资产风险成本：`http://localhost:3001/indicator-data/ASSET/RISK`

## 数据关联关系

```
indicator_summary (指标汇总)
    ↓ indicator_code
atomic_indicator (原子指标配置)
    ↓ source_table + source_field
biz_ledger (业务明细台账)

cost_allocation_result (费用分摊结果)
    ↓ cost_code
cost_type_config (费用类型配置)
    ↓ parent_code
atomic_indicator (原子指标配置)
```

## 核心业务逻辑

### 运营成本 = 各类费用之和

**资产运营成本（OP_COST）= 数据使用费 + 催收费 + 系统使用费**
- 2026-01: 954.20 + 530.00 + 242.70 = **1,726.90万元**

**负债运营成本（LIABILITY_OP_COST）= 柜面服务费 + 账户管理费**
- 2026-01: 791.60 + 395.60 = **1,187.20万元**

### 数据展示逻辑

1. **指标数据页面**：展示运营成本汇总值
2. **点击运营成本**：显示费用类型列表（从 cost_type_config 获取）
3. **点击费用类型**：显示该费用的分摊结果明细（从 cost_allocation_result 获取）
4. **点击费用类型**：显示原始费用数据（从 cost_actual_record 获取）

## 注意事项

1. 测试数据基于2026年1月账期
2. 金额单位均为万元
3. 费用分摊数据包含分摊来源维度和目标账户
