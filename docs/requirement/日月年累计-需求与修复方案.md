# 驾驶舱/维度分析 —— 当日·当月累计·当年累计 修复方案

> 日期：2026-07-01　状态：待确认

## 一、需求（用户确认）

1. 数据全部**模拟**，但要模拟**最底层数据**，然后**一步步计算、逐层汇总**。
2. 每天的利息按 **`日利息 = 余额 × 年化利率 ÷ 365`** 计算。
3. 驾驶舱与维度分析展示三个口径：**当日 / 当月累计 / 当年累计**。

## 二、现状核实（已查数据库）

### 好消息：底层表结构本就是按你的思路设计的
ODS 层 `loan_indicator_detail` / `deposit_indicator_detail` 已含全部要素字段：
- `stat_date`（业务日期）、`loan_balance`/`deposit_balance`（余额）
- `loan_rate`/`deposit_rate`（**年化利率**）、`ftp_rate`（FTP利率）
- `loan_daily_interest`（日息）、`loan_monthly_interest`（月息）、`loan_cumulative_interest`（累计息）
- 校验：余额×年化÷365 ≈ 实际日息，公式吻合。

### 三个根本问题（导致页面数据空 / 只显示单月）
| # | 问题 | 后果 |
|---|------|------|
| P0-1 | ODS 只有 18个月 × 每月1天（15号）数据，**不是逐日** | "当日""累计"无法真实计算 |
| P0-2 | DWS 表 `dw_indicator_fact` 只有 `period_type='MONTH'`，无 DAY/YEAR | 驾驶舱"本年"实为查单月 |
| P0-3 | 驾驶舱/维度查询按中文名 `dim_name` 匹配，且 `总行`(id=1) 与下级 `parent_id=93/94` 断裂、两表 collation 冲突 | 维度页大面积空、机构树顶层为0 |
| P1 | 后端只返回4个KPI，前端要8个；交叉钻取方法是空壳 | 卡片/成本项空、交叉分析无数据 |

## 三、修复方案（自底向上，五步）

### 第1步：重造最底层逐日模拟数据（ODS）
- 保留现有维度主数据（机构/产品/条线/渠道/客户经理）。
- 为每笔业务在其存续期内**逐日生成**记录（`stat_date` 覆盖每一天，而非每月15号）：
  - 每笔业务给定：初始余额、年化利率（`loan_rate`/`deposit_rate`）、FTP利率。
  - 余额按日小幅波动（模拟真实变动）。
  - `日利息 = 余额 × 年化 ÷ 365`；`累计利息 = 期初至当日日息之和`。
- 数据量控制：建议每日约 200~500 笔 × 约 540 天，用存储过程批量生成（可先跑近 3 个月验证，再全量）。

### 第2步：DWD 清洗层增加日粒度
- `dwd_loan_detail`/`dwd_deposit_detail` 增加 `stat_date` 字段，保留逐日明细并关联维度主数据（用 **ID 关联**，不用中文名）。

### 第3步：DWS 汇总层支持 DAY / MONTH / YEAR
- `dw_indicator_fact` 的 `period_type` 扩展为 `DAY / MONTH / YEAR` 三种：
  - DAY：当日各指标
  - MONTH：当月1号→当日累计
  - YEAR：当年1月1号→当日累计
- 汇总同时输出 `TOTAL` 及各维度（ORG/PRODUCT/...），**维度用 code 关联**。
- 改造 ETL 存储过程 `sp_etl_recalculate` / `sp_etl_batch` 支持三种周期。

### 第4步：后端查询逻辑修正
- `DashboardServiceImpl` / `DimensionServiceImpl`：
  - 根据前端选择的口径查对应 `period_type`（当日=DAY，当月累计=MONTH，当年累计=YEAR）。
  - KPI 卡补齐至前端所需 8 个：总利润/贷款利润/存款利润/贷款收入/存款收入/FTP成本/风险成本/运营成本，名称与前端 `getCard('...')` 完全对齐。
  - 维度查询改用维度 **code/id 关联**，修复 collation 冲突。
- 实现空壳方法：`crossDrillTree` / `getDrillPath` / `crossDrill`。

### 第5步：前端三口径切换
- 驾驶舱顶部改为「当日 / 当月累计 / 当年累计」三选项（替换现"今日/本月/本年"），传对应口径参数。
- 维度分析页同步。

## 四、修复顺序与验收
1. 第1步（造数）→ 抽查：某业务逐日余额×年化÷365 = 日息，月累计=当月日息和。
2. 第2~3步（ETL）→ 抽查：DWS 的 YEAR 累计 = 该年 DAY 之和。
3. 第4~5步（前后端）→ 页面三口径均有数、维度树含各级机构、交叉分析可用。

## 五、待确认项

- 是否保留旧 `biz_ledger` 表（建议以 ODS 明细为唯一事实源，`biz_ledger` 停用或由 ODS 重建）。
