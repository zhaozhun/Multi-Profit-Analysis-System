# 全维度亏损扫描技能

## 功能说明
扫描各维度下的亏损情况，找出问题点

## 扫描维度
- 机构维度：扫描各分行/支行的亏损情况
- 产品维度：扫描各产品线的亏损情况
- 部门维度：扫描各部门的亏损情况

## 输出格式
```
📊 全维度亏损扫描报告

【扫描期间】2026年6月

【机构维度亏损】
• 亏损机构数：3
• 深圳分行：-1,250万
• 杭州分行：-890万
• 成都分行：-520万

【产品维度亏损】
• 亏损产品数：2
• 信用卡业务：-1,850万
• 消费贷款：-680万

【部门维度亏损】
• 亏损部门数：1
• 零售部：-1,200万

【总结】
• 共发现 6 个亏损点
• 建议重点关注亏损最严重的维度，分析原因并制定改善措施
```

## 使用场景
- 用户询问"哪些机构亏损"
- 用户要求"扫描亏损情况"
- 用户想了解"低利润产品"
