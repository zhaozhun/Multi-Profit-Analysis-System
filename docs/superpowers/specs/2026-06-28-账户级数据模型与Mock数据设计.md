# 账户级数据模型与Mock数据设计

## 1. 背景

当前系统中，维度分析（机构、条线、产品等）需要基于账户级数据进行汇总。用户希望：

1. 不同类型的运营费用分摊到账户级
2. 账户级数据包含：利息收支、FTP收支、运营成本、风险成本
3. 账户级数据与维度关联
4. 按维度汇总得到维度分析结果

## 2. 数据模型设计

### 2.1 账户级数据（biz_ledger）

**核心思路**：biz_ledger 的每一行 = 一个账户级数据（一条贷款/存款记录）

**已有字段**：
- `biz_id`：业务ID（账户级唯一标识）
- `stat_date`：统计日期
- `account_period`：账期月份
- 维度字段：org_id、product_id、biz_line_id、dept_id、channel_id、manager_id、customer_id
- 金额字段：interest_income、interest_expense、ftp_cost、risk_cost、op_cost、net_profit

**运营成本关联**：
- `op_cost` = 各类费用之和（汇总值）
- 费用明细通过 `cost_allocation_result` 表关联
- 关联条件：cost_allocation_result.target_code = biz_ledger.biz_id

### 2.2 费用分摊关联

**分摊流程**：
```
费用原始数据（cost_actual_record）
  ↓ 分摊规则
费用分摊结果（cost_allocation_result）
  ↓ target_code = biz_id
账户级数据（biz_ledger.op_cost）
```

**分摊结果表结构**：
- `cost_code`：费用编码
- `cost_name`：费用名称
- `target_code`：目标账户ID（biz_id）
- `target_name`：目标账户名称
- `allocated_amount`：分摊金额
- `allocation_method`：分摊方法

### 2.3 维度汇总逻辑

**汇总公式**：
```sql
SELECT
    org_id,
    SUM(interest_income) as total_interest_income,
    SUM(ftp_cost) as total_ftp_cost,
    SUM(op_cost) as total_op_cost,
    SUM(risk_cost) as total_risk_cost,
    SUM(net_profit) as total_net_profit
FROM biz_ledger
WHERE account_period = '2026-01'
GROUP BY org_id
```

**支持的维度**：
- 机构（org_id）
- 条线（biz_line_id）
- 产品（product_id）
- 部门（dept_id）
- 渠道（channel_id）
- 客户经理（manager_id）

## 3. Mock数据设计

### 3.1 数据规模

| 项目 | 数量 |
|------|------|
| 账户级数据 | 1000条 |
| 时间范围 | 2025-01 到 2026-06（17个月） |
| 数据分布 | 随机分布 |

### 3.2 数据分布

**资产类（贷款）**：约700条
- 个人消费贷款：200条
- 个人经营贷款：150条
- 企业流动资金贷款：200条
- 企业固定资产贷款：150条

**负债类（存款）**：约300条
- 个人活期存款：100条
- 个人定期存款：100条
- 企业活期存款：50条
- 企业定期存款：50条

### 3.3 金额范围

**资产类**：
- 利息收入：10,000 ~ 500,000 元
- FTP成本：5,000 ~ 200,000 元
- 风险成本：1,000 ~ 100,000 元
- 运营成本：2,000 ~ 80,000 元

**负债类**：
- FTP收入：5,000 ~ 200,000 元
- 对客利息支出：3,000 ~ 150,000 元
- 运营成本：1,000 ~ 50,000 元

### 3.4 费用分摊数据

**费用类型**（每月）：
- 数据使用费（DATA_FEE）
- 催收费（COLLECTION_FEE）
- 系统使用费（SYSTEM_FEE）
- 柜面服务费（COUNTER_FEE）
- 账户管理费（ACCOUNT_MGMT_FEE）

**分摊规则**：
- 每种费用分摊到10-20个账户
- 分摊方法：按部门工时、按机构业务量、按产品金额
- 分摊金额 = 原始费用金额 × 分摊比例

## 4. 实现计划

### 4.1 数据准备

1. 清空现有 biz_ledger 数据
2. 插入维度主数据（机构、条线、产品、部门、渠道、客户经理）
3. 插入1000条账户级数据
4. 插入费用类型配置
5. 插入费用分摊结果

### 4.2 数据验证

1. 验证账户级数据完整性
2. 验证费用分摊一致性（op_cost = 各类费用之和）
3. 验证维度汇总正确性

### 4.3 前端展示

1. 维度分析页面展示汇总数据
2. 点击维度可查看账户级明细
3. 费用分摊页面展示分摊结果

## 5. 验收标准

1. 1000条账户级数据成功插入
2. 费用分摊数据与账户级数据关联正确
3. 维度汇总数据与账户级数据一致
4. 前端页面正常展示
