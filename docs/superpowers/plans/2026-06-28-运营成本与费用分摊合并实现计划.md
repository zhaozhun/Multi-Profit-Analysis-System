# 运营成本与费用分摊合并实现计划

## 1. 任务概述

将"费用分摊"页面功能合并到"指标数据 → 运营成本"页面，采用三个Tab页结构，体现"运营成本 = 各类费用之和"的关系和分摊逻辑。

## 2. 实现步骤

### 步骤1：修改前端页面结构

**目标**：将运营成本页面改为Tab页结构

**文件**：`前端/src/pages/BaseData/IndicatorData/index.tsx`

**修改内容**：
1. 引入Tabs组件
2. 将页面内容分为三个Tab：
   - Tab1：指标汇总（运营成本 + 费用构成）
   - Tab2：费用分摊（分摊结果 + 执行分摊）
   - Tab3：分摊配置（分摊规则、因子配置）

**预计时间**：2小时

### 步骤2：实现Tab1 - 指标汇总

**目标**：展示运营成本汇总和费用构成

**修改内容**：
1. 保留现有的"指标汇总"卡片
2. 增加"费用构成"展示区域：
   - 显示各费用类型的金额
   - 显示合计金额
   - 体现"运营成本 = 各类费用之和"的关系

**预计时间**：1小时

### 步骤3：实现Tab2 - 费用分摊

**目标**：展示费用分摊结果，支持执行分摊

**修改内容**：
1. 顶部：费用类型选择器（下拉框）
2. 中部：当前费用的分摊规则和因子展示
3. 底部：分摊结果明细表格
4. 增加"执行分摊"按钮

**预计时间**：2小时

### 步骤4：实现Tab3 - 分摊配置

**目标**：展示和编辑分摊规则、因子配置

**修改内容**：
1. 费用类型列表（表格）
2. 每个费用的分摊规则、因子配置
3. 支持编辑分摊规则和因子

**预计时间**：2小时

### 步骤5：修改菜单配置

**目标**：删除"费用分摊"菜单入口

**文件**：`前端/src/components/MainLayout.tsx`

**修改内容**：
1. 删除"费用分摊"菜单项
2. 保留"指标数据 → 运营成本"菜单项

**预计时间**：0.5小时

### 步骤6：增加后端API

**目标**：支持执行分摊和分摊配置管理

**文件**：后端API

**修改内容**：
1. 增加"执行分摊"API：`POST /api/indicator/execute-allocation`
2. 增加"获取分摊配置"API：`GET /api/indicator/allocation-config`
3. 增加"更新分摊配置"API：`PUT /api/indicator/allocation-config`

**预计时间**：2小时

### 步骤7：测试和验收

**目标**：确保功能正常

**测试内容**：
1. 运营成本页面展示三个Tab页
2. Tab1正确展示运营成本汇总和费用构成
3. Tab2正确展示费用分摊结果，支持选择费用类型
4. Tab3正确展示分摊配置，支持编辑
5. "执行分摊"按钮功能正常
6. 资产和负债页面独立展示
7. 删除"费用分摊"菜单入口

**预计时间**：1小时
