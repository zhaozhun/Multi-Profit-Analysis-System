# 指标数据优化设计文档

> 设计日期：2026-06-29
> 设计状态：已确认

---

## 一、设计背景

### 1.1 问题描述

当前系统存在以下问题：
1. 贷款数据缺少利率、当日利息、当月利息、累计利息等关键字段
2. 存款数据同样缺少付息率和分层利息字段
3. 指标数据页面和维度分析页面的边界不清晰

### 1.2 设计目标

1. 扩展数据模型，支持利率和分层利息数据
2. 明确指标数据和维度分析两个模块的边界
3. 优化指标数据页面，支持按维度筛选和明细展示

---

## 二、模块边界划分

### 2.1 指标数据层（预计算）

**目的**：提前算好各个指标，存储计算结果

**职责**：
- 预计算各个指标（利息收入、FTP成本、风险成本、运营成本）
- 按7个维度（机构/条线/部门/产品/渠道/客户经理/客户）分解
- 提供汇总和明细查询

**典型问题**：
- 利息收入是多少？
- 平均利率是多少？
- 按机构分解的利息收入是多少？

### 2.2 维度分析层（直接取数）

**目的**：从预计算结果中直接取数，计算利润

**职责**：
- 直接读取指标预计算结果
- 按维度汇总利润（收入 - 成本）
- 提供排名、交叉分析、下钻等功能

**典型问题**：
- 哪个支行利润最高？
- 某机构的利润构成是什么？

### 2.3 数据流向

```
┌─────────────────────────────────────────────────────────────────┐
│                         指标数据层                               │
│                    （预计算，存储结果）                           │
│                                                                 │
│   利息收入指标 ─────────────────────────────────────────────┐   │
│   │ 按机构：浦西支行 150,000 / 天河支行 120,000 / ...       │   │
│   │ 按产品：流贷 500,000 / 固贷 300,000 / ...               │   │
│   └───────────────────────────────────────────────────────┘   │
│                                                                 │
│   FTP成本指标 ──────────────────────────────────────────────┐   │
│   │ 按机构：浦西支行 60,000 / 天河支行 48,000 / ...          │   │
│   └───────────────────────────────────────────────────────┘   │
│                                                                 │
│   风险成本指标 ─────────────────────────────────────────────┐   │
│   │ 按机构：浦西支行 15,000 / ...                            │   │
│   └───────────────────────────────────────────────────────┘   │
│                                                                 │
│   运营成本指标 ─────────────────────────────────────────────┐   │
│   │ 按机构：浦西支行 10,000 / ...                            │   │
│   └───────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
                              │
                              │ 直接取数
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                         维度分析层                               │
│                    （取预计算结果，算利润）                       │
│                                                                 │
│   浦西支行利润 = 利息收入 - FTP成本 - 风险成本 - 运营成本         │
│               = 150,000 - 60,000 - 15,000 - 10,000             │
│               = 65,000                                          │
└─────────────────────────────────────────────────────────────────┘
```

## 三、业务逻辑

### 3.1 贷款业务（资产）

**利润计算公式**：
```
贷款利润 = 贷款收入（利息收入） - FTP成本 - 风险成本 - 运营成本
```

**字段说明**：
| 字段 | 说明 | 来源 |
|------|------|------|
| 贷款收入 | 利息收入 | biz_ledger.loan_revenue |
| FTP成本 | 资金转移定价成本 | biz_ledger.loan_ftp_cost |
| 风险成本 | 风险计提成本 | biz_ledger.loan_risk_cost |
| 运营成本 | 运营费用分摊 | biz_ledger.loan_op_cost |
| 贷款利润 | 收入 - 成本 | biz_ledger.loan_profit |

### 3.2 存款业务（负债）

**利润计算公式**：
```
存款利润 = 存款收入（FTP收入） - 存款利息支出 - 运营成本
```

**字段说明**：
| 字段 | 说明 | 来源 |
|------|------|------|
| 存款收入 | FTP收入（内部资金转移定价收入） | biz_ledger.deposit_revenue |
| 存款利息支出 | 支付给客户的利息 | biz_ledger.deposit_interest |
| 运营成本 | 运营费用分摊 | biz_ledger.deposit_op_cost |
| 存款利润 | 收入 - 成本 | biz_ledger.deposit_profit |

**关键区别**：
- 贷款：有FTP成本、风险成本、运营成本
- 存款：没有FTP成本、没有风险成本，只有利息支出和运营成本

### 3.3 利息计算逻辑

**计算公式**：
- 当日利息 = 余额 × 利率 / 365
- 当月利息 = 当日利息 × 当月已过天数
- 累计利息 = 当日利息 × 贷款已存续天数

**利率范围**：
- 贷款利率：3% - 15%
- 存款付息率：1% - 3%

## 四、数据库设计

### 4.1 设计方案：混合模式

**主表（biz_ledger）**：存储当前/最新利率，用于实时查询
**历史表（interest_rate_history）**：存储利率变化，用于历史回溯

### 4.2 biz_ledger 表扩展字段

```sql
-- 贷款相关字段
ALTER TABLE biz_ledger ADD COLUMN loan_rate DECIMAL(8,4) COMMENT '贷款利率(%)';
ALTER TABLE biz_ledger ADD COLUMN loan_daily_interest DECIMAL(18,2) COMMENT '贷款当日利息收入';
ALTER TABLE biz_ledger ADD COLUMN loan_monthly_interest DECIMAL(18,2) COMMENT '贷款当月利息收入';
ALTER TABLE biz_ledger ADD COLUMN loan_cumulative_interest DECIMAL(18,2) COMMENT '贷款累计利息收入';

-- 存款相关字段
ALTER TABLE biz_ledger ADD COLUMN deposit_rate DECIMAL(8,4) COMMENT '存款付息率(%)';
ALTER TABLE biz_ledger ADD COLUMN deposit_daily_interest DECIMAL(18,2) COMMENT '存款当日利息支出';
ALTER TABLE biz_ledger ADD COLUMN deposit_monthly_interest DECIMAL(18,2) COMMENT '存款当月利息支出';
ALTER TABLE biz_ledger ADD COLUMN deposit_cumulative_interest DECIMAL(18,2) COMMENT '存款累计利息支出';
```

### 4.3 利率历史表

```sql
CREATE TABLE interest_rate_history (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  biz_id VARCHAR(30) NOT NULL COMMENT '业务ID',
  product_type VARCHAR(20) NOT NULL COMMENT '产品类型(LOAN/DEPOSIT)',
  rate_date DATE NOT NULL COMMENT '利率日期',
  interest_rate DECIMAL(8,4) NOT NULL COMMENT '当日利率(%)',
  daily_interest DECIMAL(18,2) COMMENT '当日利息',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  
  INDEX idx_biz_date (biz_id, rate_date),
  INDEX idx_product_date (product_type, rate_date)
) COMMENT '利率历史快照表';
```

### 4.4 数据关系

```
biz_ledger (主表)
├── loan_rate ──────────────► 当前贷款利率
├── loan_daily_interest ───► 当日利息（实时）
├── loan_monthly_interest ─► 当月累计（聚合）
├── loan_cumulative_interest► 历史累计（聚合）
│
└── interest_rate_history (历史表)
    ├── biz_id ────────────► 关联主表
    ├── rate_date ─────────► 利率日期
    ├── interest_rate ─────► 当日利率
    └── daily_interest ────► 当日利息
```

### 4.5 模拟数据生成规则

**贷款数据**：
```sql
UPDATE biz_ledger SET
  loan_rate = 3 + RAND() * 12,
  loan_daily_interest = biz_amount * (3 + RAND() * 12) / 100 / 365,
  loan_monthly_interest = biz_amount * (3 + RAND() * 12) / 100 / 12,
  loan_cumulative_interest = biz_amount * (3 + RAND() * 12) / 100 * (1 + RAND() * 2)
WHERE product_type = 'LOAN';
```

**存款数据**：
```sql
UPDATE biz_ledger SET
  deposit_rate = 1 + RAND() * 2,
  deposit_daily_interest = biz_amount * (1 + RAND() * 2) / 100 / 365,
  deposit_monthly_interest = biz_amount * (1 + RAND() * 2) / 100 / 12,
  deposit_cumulative_interest = biz_amount * (1 + RAND() * 2) / 100 * (1 + RAND() * 2)
WHERE product_type = 'DEPOSIT';
```

## 五、前端页面设计

### 5.1 设计原则

1. **驾驶舱页面**：维持现状，不改
2. **维度分析页面**：保持现有风格，只增加利息字段
3. **指标数据页面**：保持左侧栏菜单，每个指标独立页面

### 5.2 菜单结构（保持不变）

```
指标数据
├── 资产
│   ├── 利息收入    ◄── 独立页面
│   ├── FTP成本     ◄── 独立页面
│   ├── 风险成本    ◄── 独立页面
│   └── 运营成本    ◄── 独立页面
└── 负债

```


## 八、风险与注意事项

### 8.1 数据一致性
- 确保指标预计算结果与维度分析取数一致
- 确保利息收入 = 当月利息汇总

### 8.2 性能考虑
- 预计算可以提高查询性能
- 历史表数据量大，需要定期归档

### 8.3 数据准确性
- 利率范围需要符合实际业务
- 利息计算逻辑需要验证
