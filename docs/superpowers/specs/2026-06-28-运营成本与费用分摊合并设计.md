# 运营成本与费用分摊合并设计

## 1. 背景

当前系统中存在三个相关页面：
- 资产运营成本页面 `/indicator-data/ASSET/OP`
- 负债运营成本页面 `/indicator-data/LIABILITY/OP`
- 运营费用分摊页面 `/allocation/operation-cost`

这些页面数据有重叠，关系不清晰。用户希望：
1. 统一这三个页面的设计
2. 体现"运营成本 = 各类费用之和"的关系
3. 体现分摊逻辑

## 2. 设计方案

### 2.1 页面合并

将"费用分摊"页面功能合并到"指标数据 → 运营成本"页面，删除"费用分摊"菜单入口。

### 2.2 Tab页结构

合并后的运营成本页面采用三个Tab页结构：

**Tab1：指标汇总**
- 运营成本汇总（1,726.90万）
- 费用构成展示：
  ```
  数据使用费 954.20万
  催收费 530.00万
  系统使用费 242.70万
  ─────────────────
  合计 1,726.90万
  ```

**Tab2：费用分摊**
- 顶部：费用类型选择器（下拉框）
- 中部：当前费用的分摊规则和因子展示
- 底部：分摊结果明细表格

**Tab3：分摊配置**
- 费用类型列表（表格）
- 每个费用的分摊规则、因子配置
- 支持编辑分摊规则和因子

### 2.3 业务条线分开

资产和负债各自有独立的运营成本页面：
- 资产运营成本：`/indicator-data/ASSET/OP`
- 负债运营成本：`/indicator-data/LIABILITY/OP`

### 2.4 执行分摊功能

在Tab2（费用分摊）中增加"执行分摊"按钮，支持手动触发分摊计算。

## 3. 交互流程

1. 用户在Tab1看到运营成本的构成
2. 切换到Tab2，选择具体费用类型，查看分摊结果
3. 切换到Tab3，配置或修改分摊规则和因子

## 4. 涉及页面

### 4.1 前端页面

- `/indicator-data/ASSET/OP`：资产运营成本页面
- `/indicator-data/LIABILITY/OP`：负债运营成本页面

### 4.2 菜单配置

- `MainLayout.tsx`：删除"费用分摊"菜单入口

### 4.3 后端API

- `GET /api/indicator/summary`：获取指标汇总数据
- `GET /api/indicator/cost-types`：获取费用类型列表
- `GET /api/indicator/cost-allocation`：获取费用分摊结果
- `POST /api/indicator/execute-allocation`：执行分摊（新增）

## 5. 数据结构

### 5.1 费用类型配置表 (cost_type_config)

| 字段 | 说明 | 示例 |
|------|------|------|
| cost_type_code | 费用类型编码 | DATA_FEE |
| cost_type_name | 费用类型名称 | 数据使用费 |
| parent_code | 父级编码 | OP_COST |
| default_algorithm | 默认算法 | BY_DEPT_RATIO |
| default_factor | 默认因子 | WORK_HOUR |
| status | 状态 | ACTIVE |

### 5.2 费用分摊结果表 (cost_allocation_result)

| 字段 | 说明 | 示例 |
|------|------|------|
| cost_code | 费用编码 | DATA_FEE |
| cost_name | 费用名称 | 数据使用费 |
| target_code | 目标编码 | ACC001 |
| target_name | 目标名称 | 账户1 |
| allocated_amount | 分摊金额 | 125.50 |
| allocation_method | 分摊方法 | BY_DEPT_RATIO |
