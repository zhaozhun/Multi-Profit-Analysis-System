# 部署指南

## 快速部署（推荐）

### 第一步：服务器执行环境安装脚本

```bash
# SSH 登录你的火山云服务器
ssh root@你的服务器IP

# 上传部署脚本
scp deploy/deploy.sh root@你的服务器IP:/tmp/

# 执行部署脚本（安装 Java、MySQL、Nginx 等）
bash /tmp/deploy.sh
```

### 第二步：本地打包上传

```bash
# 修改 deploy/upload.sh 中的 SERVER_IP 为你的服务器 IP
# 然后执行
bash deploy/upload.sh
```

### 第三步：访问系统

浏览器打开 `http://你的服务器IP`

## 手动部署

如果自动脚本有问题，按以下步骤手动操作：

### 1. 服务器安装依赖

```bash
# Ubuntu/Debian
apt update
apt install -y openjdk-17-jdk nginx mysql-server

# CentOS/RHEL
yum install -y java-17-openjdk nginx mysql-server
```

### 2. 配置数据库

```bash
mysql -u root -p
```
```sql
CREATE DATABASE multi_profit CHARACTER SET utf8mb4;
CREATE USER 'mpuser'@'localhost' IDENTIFIED BY '你的密码';
GRANT ALL ON multi_profit.* TO 'mpuser'@'localhost';
FLUSH PRIVILEGES;

USE multi_profit;
SOURCE /tmp/create-tables.sql;
SOURCE /tmp/master-data-3level.sql;
```

### 3. 上传并启动后端

```bash
# 本地打包
cd backend && mvn clean package -DskipTests

# 上传
scp target/multi-profit-analysis-1.0.0-SNAPSHOT.jar root@服务器IP:/opt/multiprofit/multi-profit-analysis.jar

# 服务器上创建配置
cat > /opt/multiprofit/application-prod.yml << 'EOF'
server:
  port: 8080
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/multi_profit?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: mpuser
    password: 你的密码
    driver-class-name: com.mysql.cj.jdbc.Driver
EOF

# 创建 systemd 服务
cat > /etc/systemd/system/multiprofit.service << 'EOF'
[Unit]
Description=Multi Profit Analysis
After=network.target mysql.service

[Service]
ExecStart=/usr/bin/java -jar /opt/multiprofit/multi-profit-analysis.jar --spring.config.location=/opt/multiprofit/application-prod.yml
Restart=always

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable multiprofit
systemctl start multiprofit
```

### 4. 上传并配置前端

```bash
# 本地打包（修改 API 地址）
cd frontend
echo "VITE_API_BASE=http://服务器IP/api" > .env.production
npm run build

# 上传
scp -r dist/* root@服务器IP:/var/www/multiprofit/

# 配置 Nginx（见 deploy.sh 中的配置）
```

## 常见问题

### Q: 访问 404？
A: 检查 Nginx 配置和前端文件是否在正确目录

### Q: API 请求失败？
A: 检查后端是否启动 `systemctl status multiprofit`

### Q: 数据库连接失败？
A: 检查 MySQL 是否运行，用户名密码是否正确

### Q: 端口不通？
A: 火山云控制台 → 安全组 → 放行 80 和 8080 端口

---

## 火山云安全组配置

1. 登录火山云控制台
2. 找到你的服务器实例
3. 进入「安全组」设置
4. 添加入站规则：
   - 协议: TCP, 端口: 80, 来源: 0.0.0.0/0
   - 协议: TCP, 端口: 8080, 来源: 0.0.0.0/0
   - 协议: TCP, 端口: 443, 来源: 0.0.0.0/0
