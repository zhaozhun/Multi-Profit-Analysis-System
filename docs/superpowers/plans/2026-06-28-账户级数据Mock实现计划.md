### 步骤5：生成费用分摊数据

**目标**：为每月每种费用生成分摊结果

**分摊逻辑**：
1. 获取当月原始费用金额
2. 按分摊规则计算分摊比例
3. 生成分摊结果

**预计时间**：1小时

### 步骤6：更新账户级运营成本

**目标**：将分摊结果汇总到 biz_ledger.op_cost

**SQL**：
```sql
UPDATE biz_ledger bl
SET op_cost = (
    SELECT COALESCE(SUM(car.allocated_amount), 0)
    FROM cost_allocation_result car
    WHERE car.target_code = bl.biz_id
    AND car.period = bl.account_period
)
WHERE bl.account_period IN (SELECT DISTINCT period FROM cost_allocation_result);
```

**预计时间**：0.5小时

### 步骤7：数据验证

**目标**：验证数据完整性和一致性

**验证内容**：
1. 账户级数据数量：1000条
2. 费用分摊数据完整性
3. 运营成本一致性（op_cost = 各类费用之和）
4. 维度汇总正确性

**预计时间**：0.5小时

## 4. 风险和依赖

### 4.1 风险

1. **数据量大**：1000条数据需要批量插入，可能影响性能
2. **分摊逻辑复杂**：需要确保分摊金额计算正确
3. **数据一致性**：需要确保运营成本与费用分摊一致

### 4.2 依赖

1. 数据库表结构已存在
2. 维度主数据表结构正确
3. 费用分摊结果表结构正确

## 5. 验收标准

2. 费用分摊数据与账户级数据关联正确
