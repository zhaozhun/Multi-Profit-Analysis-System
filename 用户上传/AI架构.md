# 多维盈利分析系统 AI 能力集成全方案
在盈利分析场景中，**Function Call 是原子工具、Skills 是标准化业务能力包、Agent 是自主分析执行者、MCP 是统一接入总线**，四者分层协作而非互斥，共同构建「数据→洞察→决策」的全链路智能化体系。以下结合财务分析的行业落地实践，拆解具体集成方式与能力边界。

---

## 一、整体分层架构
从底层到交互层，能力逐级封装，职责清晰解耦：

| 层级 | 技术载体 | 核心职责 |
|------|----------|----------|
| 数据源层 | ERP / CRM / 数仓 / 预算系统 | 提供原始业务与财务数据 |
| 协议接入层 | MCP Server 集群 | 标准化封装数据、算法、权限，统一暴露能力 |
| 工具原子层 | Function Call | 单一职责的原子操作，如查数、计算、导出 |
| 业务技能层 | Skills | 组合多个工具，完成固定流程的标准化分析任务 |
| 智能执行层 | Agent | 自主拆解任务、规划路径、多步推理，处理复杂分析 |
| 交互入口层 | Web平台 / BI看板 / 企业IM | 用户自然语言交互、可视化呈现 |

---

## 二、Function Call：底层原子能力，所有智能化的基石
Function Call 是大模型调用外部系统的基础入口，**只做单一动作，不做业务编排**，输入输出严格结构化，保证数据口径唯一、计算可追溯。

### 可落地的核心函数分类
1. **数据查询类（数据口径统一收口）**
    - `query_profit_metrics(dimensions, metrics, filters)`：按产品/客户/区域/渠道等任意维度组合，查询收入、成本、毛利、费用、净利润等核心指标
    - `drill_down_detail(parent_dim, child_dim, filters)`：维度下钻，从大区→省份→客户→订单逐层穿透明细
    - `query_period_compare(period, compare_type)`：自动计算同比、环比、预算达成率及差异额
2. **计算分析类（算法逻辑统一收口）**
    - `calculate_dimension_contribution(target_metric, dimension, period)`：计算各维度分项对总利润波动的贡献度与影响占比
    - `calculate_profit_sensitivity(factor, change_range)`：单因素/多因素盈利敏感性测算
    - `allocate_overhead_cost(rule_id, dimensions)`：按预设规则分摊间接费用，保证口径一致
3. **操作输出类**
    - `generate_report(template_id, period)`：按模板生成标准化报表
    - `trigger_metric_alert(metric, threshold, receiver)`：触发指标异常预警推送

### 设计原则
- 一个函数只做一件事，不掺杂业务判断逻辑
- 所有返回结果附带数据口径、计算规则、时间范围，全程可溯源
- 统一收口所有数据访问，避免AI直接操作数据库带来的安全与口径风险

## 三、Skills：标准化业务能力包，高频分析开箱即用
Skill 是**多个 Function Call 按固定业务流程组合成的完整能力单元**，流程固定、结果可预期，一次封装全系统复用，对应财务分析中的标准化场景。

### 核心盈利分析 Skill 清单
1. **利润波动归因 Skill**
    - 内部流程：拉取基期+报告期数据 → 按维度拆解贡献度 → 排序核心驱动因素 → 生成结构化结论
    - 输入：分析周期、对比基准、指定分析维度
    - 输出：完整归因报告，包含核心影响因素、贡献金额、影响占比、数据依据
    - 价值：替代人工手动拆解波动，将数小时的工作压缩至分钟级

2. **全维度亏损扫描 Skill**
    - 内部流程：遍历所有维度组合 → 计算细分项利润率 → 筛选亏损/低利项 → 匹配亏损原因标签（成本过高、定价偏低、费用分摊过重等）
    - 输出：亏损清单 + 严重程度排序 + 初步原因定位
    - 参考实践：金蝶AI盈利分析的异常洞察能力，自动标记偏离历史阈值的异常项

3. **经营分析报告生成 Skill**
    - 内部流程：拉取核心指标 → 维度拆解数据 → 生成图表 → 撰写文字洞察 → 按模板排版
    - 支持周/月/季度自动生成，可直接导出 PPT/PDF，附带可复核的数据来源

4. **盈利情景模拟 Skill**
    - 内部流程：接收变动参数 → 调用测算函数 → 计算多情景利润结果 → 生成敏感性对比表
    - 支持原材料涨价、产品提价、费用缩减等多因素组合测算，输出量化影响结果

5. **客户/产品盈利分层 Skill**
    - 内部流程：计算单体盈利性 → 按利润贡献分层 → 分析各层特征 → 输出优化方向标签

### 设计原则
- 面向固定业务场景，流程可标准化，不做开放式推理
- 输入输出结构清晰，可直接嵌入BI看板或定时任务
- 所有计算逻辑复用底层 Function Call，保证全系统口径一致

## 四、Agent：自主分析执行者，处理复杂非标准化任务
Agent 具备**任务拆解、自主规划、多步推理、工具调用、结论迭代**的能力，适合没有固定流程的开放式分析场景，是从「人找数」到「数找人」的核心载体。

### 四类核心盈利分析 Agent
1. **专项分析 Agent（核心价值最高）**
    - 适用场景：开放式分析问题，例如「Q3利润未达标，核心问题出在哪？」
    - 自主执行链路：
      1. 目标拆解：先确认整体利润缺口，确定分析框架（收入端/成本端/费用端）
      2. 路径规划：先按大区维度排查，定位异常区域后自动下钻至产品、客户维度
      3. 交叉验证：从业务动因和财务结果双向验证，排除偶发因素
      4. 结论输出：汇总根因，附上数据依据和可落地的业务建议
    - 参考实践：金蝶财务分析智能体，实现从「是什么」到「为什么」的穿透分析

2. **监控预警 Agent**
    - 定时巡检核心盈利指标（毛利率、净利率、费用率），偏离历史阈值自动触发
    - 发现异常后自动启动根因分析，生成预警单推送给对应业务负责人
    - 持续跟踪异常指标改善情况，形成「发现-分析-跟进-闭环」全链路

3. **业务优化顾问 Agent**
    - 基于全量盈利数据，主动识别盈利提升机会
    - 输出具体建议：哪些产品应提价、哪些客户应优化服务成本、哪些费用有压缩空间
    - 附带量化收益预估，辅助决策判断

4. **自助答疑 Agent**
    - 承接业务人员的自然语言查询，例如「我负责的华南区本月A产品毛利率是多少」
    - 自动转译查询、取数、解读结果，降低数据使用门槛，释放财务人力

### 设计原则
- Agent 只做规划与判断，所有数据操作和计算全部调用底层工具，不自行计算
- 保留完整分析链路日志，每一步结论都可追溯到对应数据
- 关键结论设置置信度，低置信度结论主动提示人工复核

## 五、MCP：标准化能力总线，解耦AI与业务系统
MCP（Model Context Protocol）是整个系统的**能力接入标准**，把所有数据源、工具、技能按统一协议封装，解决传统集成中接口重复开发、权限分散、口径不一的问题。微软、Databricks 等厂商已在财务分析场景落地 MCP 标准。

### 落地方式：按领域拆分 MCP Server
1. **业财数据 MCP Server**
    - 对接 ERP、数仓、CRM 等系统，统一暴露所有盈利数据查询工具
    - 内置维度模型、指标定义、表关系，AI 只需要描述需求，自动生成查询
    - 参考实践：Dynamics 365 ERP Analytics MCP，将业务数据模型通过标准协议暴露给AI，原生支持自然语言分析

2. **分析算法 MCP Server**
    - 封装归因、分摊、预测、敏感性分析等所有计算类能力
    - 统一算法逻辑和分摊规则，避免不同模块计算结果不一致

3. **权限与审计 MCP Server**
    - 统一处理用户鉴权、行级列级数据权限、字段脱敏
    - 所有AI操作全程留痕、可审计，满足财务数据合规要求

4. **上下文与记忆 MCP Server**
    - 统一管理分析会话的记忆、上下文压缩、历史归档
    - 支持 Agent 跨会话承接分析任务，例如上月分析过的异常问题，本月可自动跟踪改善情况

### MCP 带来的核心价值
- **解耦**：AI 层不需要感知底层系统架构，业务系统改造不影响AI逻辑
- **复用**：同一套能力可同时供给 Web 平台、BI 助手、企业IM 助手，无需重复对接
- **安全**：所有数据访问统一收口，权限、脱敏、审计一站式管控
- **扩展**：新增数据源或分析能力时，仅需新增一个 MCP Server，不改动核心架构

## 六、分阶段落地路径（从易到难，快速见效）
### 第一阶段：基础智能化（1-2个月）
- 核心目标：快速释放基础人力，验证价值
- 落地内容：
  1. 封装核心数据查询与指标计算 Function Call
  2. 上线「利润波动归因」「报告自动生成」两个高频 Skill
  3. 实现自然语言查数 + 月度经营报告自动生成

### 第二阶段：标准化 + Agent 落地（2-3个月）
- 核心目标：能力标准化，升级主动分析
- 落地内容：
  1. 按领域拆分 MCP Server，统一接入与权限管控
  2. 上线专项分析 Agent，支持开放式问题下钻分析
  3. 上线监控预警 Agent，自动巡检异常指标

### 第三阶段：全链路智能化（3-6个月）
- 核心目标：从事后分析升级为全周期盈利管理
- 落地内容：
  1. 上线盈利预测、情景模拟、敏感性分析能力
  2. 上线业务优化顾问 Agent，输出可落地的盈利提升建议
  3. 打通业财全链路数据，实现「预测-监控-分析-优化」闭环

## 七、财务场景关键避坑原则
1. **口径唯一优先**：所有指标定义、分摊规则必须全局统一，由底层工具层收口，严禁不同模块各自计算
2. **全程可追溯**：所有AI生成的结论必须附带数据来源、计算规则、维度口径，支持财务人员复核
3. **权限分级管控**：通过 MCP 层统一做行级、列级权限校验，不同角色仅可见授权范围内数据
4. **人工兜底机制**：重要决策建议标注「AI生成，仅供参考」，关键分析结论必须经专业人员确认
5. **严控幻觉风险**：所有数据全部通过工具调用获取，禁止大模型自行编造财务数据
